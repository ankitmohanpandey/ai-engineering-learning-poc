# AI Engineering Glossary

> **Quick reference for AI/ML terminology**

---

## A

**Agent**  
An AI system that can perceive its environment, make decisions, and take actions to achieve goals.

**AGI (Artificial General Intelligence)**  
Hypothetical AI that can understand, learn, and apply knowledge across any domain like humans.

**AI (Artificial Intelligence)**  
The simulation of human intelligence in machines programmed to think and learn.

**API (Application Programming Interface)**  
Interface that allows different software applications to communicate with each other.

**Attention Mechanism**  
Neural network component that helps models focus on relevant parts of input data.

---

## B

**Batch Processing**  
Processing multiple data samples together rather than one at a time.

**BERT (Bidirectional Encoder Representations from Transformers)**  
Google's pre-trained language model that understands context from both directions.

**Bias**  
1. Statistical: Systematic error in ML models  
2. Ethical: Unfair prejudice in AI systems

**Beam Search**  
Algorithm for finding the most likely sequence of outputs in generation tasks.

---

## C

**Chain-of-Thought (CoT)**  
Prompting technique where the model explains its reasoning step-by-step.

**Chatbot**  
AI application designed to simulate conversation with human users.

**Classification**  
ML task of categorizing data into predefined classes.

**Claude**  
Anthropic's AI assistant and large language model.

**Context Window**  
Maximum amount of text an LLM can process at once (measured in tokens).

**Corpus**  
Large collection of text used for training language models.

---

## D

**Deep Learning**  
ML using neural networks with multiple layers to learn complex patterns.

**Deployment**  
Process of making an AI model available for use in production.

**Diffusion Model**  
Type of generative model used for creating images (e.g., Stable Diffusion).

**Dimensionality Reduction**  
Reducing the number of features in data while preserving important information.

---

## E

**Embedding**  
Numerical representation of data (text, images) as vectors in high-dimensional space.

**Encoder-Decoder**  
Neural network architecture with two parts: one to process input, one to generate output.

**Epoch**  
One complete pass through the entire training dataset.

**Evaluation**  
Process of measuring model performance using metrics.

---

## F

**Few-Shot Learning**  
Model learning from just a few examples.

**Fine-Tuning**  
Adapting a pre-trained model to a specific task with additional training.

**Foundation Model**  
Large pre-trained model that can be adapted for various tasks.

**Function Calling**  
LLM capability to invoke external functions or tools.

---

## G

**Gemini**  
Google's multimodal AI model family.

**Generative AI**  
AI that creates new content (text, images, code, etc.).

**GPT (Generative Pre-trained Transformer)**  
OpenAI's language model architecture.

**Gradient Descent**  
Optimization algorithm used to train neural networks.

---

## H

**Hallucination**  
When an LLM generates plausible but incorrect or nonsensical information.

**Hyperparameter**  
Configuration setting for ML models (e.g., learning rate, temperature).

**Hugging Face**  
Platform for sharing and using ML models, especially for NLP.

---

## I

**Inference**  
Using a trained model to make predictions on new data.

**In-Context Learning**  
LLM's ability to learn from examples provided in the prompt.

**Instruction Tuning**  
Fine-tuning models to better follow instructions.

---

## J

**JSON (JavaScript Object Notation)**  
Lightweight data format commonly used for API communication.

---

## K

**K-Nearest Neighbors (KNN)**  
Simple ML algorithm that classifies based on similarity to nearby examples.

---

## L

**LangChain**  
Framework for building applications with LLMs.

**Latency**  
Time delay between input and output.

**LLM (Large Language Model)**  
AI model trained on vast amounts of text to understand and generate language.

**LlamaIndex**  
Framework for connecting LLMs with external data sources.

**Loss Function**  
Measure of how wrong a model's predictions are.

---

## M

**Machine Learning (ML)**  
Subset of AI where systems learn from data without explicit programming.

**Mem0**  
Open-source memory layer for AI applications that enables persistent memory, semantic search, and multi-user support. Helps AI agents remember information across conversations.

**Model**  
Mathematical representation learned from data to make predictions.

**Multimodal**  
AI that can process multiple types of data (text, images, audio).

---

## N

**Neural Network**  
Computing system inspired by biological neural networks in brains.

**NLP (Natural Language Processing)**  
Field of AI focused on understanding and generating human language.

**Normalization**  
Scaling data to a standard range.

---

## O

**OpenAI**  
AI research company that created GPT models and ChatGPT.

**Overfitting**  
When a model learns training data too well and performs poorly on new data.

---

## P

**Parameter**  
Learnable value in a neural network (e.g., weights, biases).

**Perplexity**  
Metric measuring how well a model predicts text.

**Prompt**  
Input text given to an LLM to generate a response.

**Prompt Engineering**  
Art and science of crafting effective prompts for LLMs.

---

## Q

**Quantization**  
Reducing model size by using lower precision numbers.

**Query**  
1. Database: Request for information  
2. Vector DB: Search for similar embeddings

---

## R

**RAG (Retrieval Augmented Generation)**  
Technique combining information retrieval with text generation.

**ReAct (Reasoning + Acting)**  
Agent pattern that alternates between reasoning and taking actions.

**Regression**  
ML task of predicting continuous numerical values.

**Reinforcement Learning**  
ML where agents learn through trial and error with rewards.

**RLHF (Reinforcement Learning from Human Feedback)**  
Training technique using human preferences to improve models.

---

## S

**Semantic Search**  
Search based on meaning rather than exact keyword matching.

**Sentiment Analysis**  
Determining emotional tone of text (positive, negative, neutral).

**Supervised Learning**  
ML using labeled training data.

**System Prompt**  
Initial instruction defining an LLM's role and behavior.

---

## T

**Temperature**  
Parameter controlling randomness in LLM outputs (0 = deterministic, 1 = creative).

**Token**  
Basic unit of text processed by LLMs (word, subword, or character).

**Tokenization**  
Process of breaking text into tokens.

**Top-k Sampling**  
Selecting from top k most likely next tokens.

**Top-p (Nucleus) Sampling**  
Selecting from smallest set of tokens whose cumulative probability exceeds p.

**Training**  
Process of teaching a model by showing it examples.

**Transfer Learning**  
Using knowledge from one task to improve performance on another.

**Transformer**  
Neural network architecture using attention mechanisms (basis for LLMs).

---

## U

**Underfitting**  
When a model is too simple to capture patterns in data.

**Unsupervised Learning**  
ML finding patterns in unlabeled data.

---

## V

**Vector**  
Array of numbers representing data in mathematical space.

**Vector Database**  
Database optimized for storing and searching vector embeddings.

**Vision Model**  
AI model that processes and understands images.

---

## W

**Weight**  
Parameter in neural networks that determines connection strength.

**Whisper**  
OpenAI's speech recognition model.

---

## Z

**Zero-Shot Learning**  
Model performing tasks without specific training examples.

---
